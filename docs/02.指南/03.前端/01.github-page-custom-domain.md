# GitHub Pages 自定义域名配置

本教程介绍如何为 GitHub Pages 站点配置自定义域名，并包含在 NameSilo 购买域名后的配置步骤。

## 1. 在 NameSilo 购买域名

1. 访问 NameSilo 官网：https://www.namesilo.com
2. 登录 NameSilo 账号，或注册一个新账号。
3. 在搜索框中输入你想要的域名，例如 `example.com`，点击搜索。
4. 选择可用的域名并加入购物车。
5. 按照步骤完成结账并付款。

> NameSilo 默认支持 DNS 管理，并提供免费隐私保护（Privacy Protection）。

## 2. 准备工作

- 确认仓库已启用 GitHub Pages。
- 准备好自有域名，例如 `example.com` 或 `www.example.com`。
- 登录 NameSilo，进入“Domain Manager”并确认你已能够修改域名的 DNS 记录。

## 4. 创建或编辑 `CNAME` 文件

`CNAME` 文件位置取决于你的 GitHub Pages 发布源：

- 如果你使用 `main` 分支的 `/docs` 文件夹作为 Pages 源，则将 `CNAME` 放到 `docs/CNAME`。
- 如果你使用 `main` 分支根目录作为 Pages 源，则将 `CNAME` 放到仓库根目录。
- 如果你使用 `gh-pages` 分支作为 Pages 源，则将 `CNAME` 放到该分支根目录。

1. 在正确的 Pages 源目录中创建一个名为 `CNAME` 的文件。
2. 文件内容只写入你的自定义域名，例如：

```text
example.com
```

3. 提交并推送该文件到 GitHub 仓库。

> 如果你使用的是根域名（裸域），`CNAME` 文件里也填写裸域名，例如 `example.com`。

## 4. 在 GitHub 仓库设置中绑定自定义域名

1. 打开你的 GitHub 仓库。
2. 进入 `Settings` -> `Pages`（或 `Environments` -> `Pages`，界面可能因 GitHub 更新而变化）。
3. 在“Custom domain”文本框中输入你的域名，例如 `www.example.com` 或 `example.com`。
4. 保存并启用 `Enforce HTTPS`（建议开启，如果域名支持 HTTPS，会自动申请 LetsEncrypt 证书）。

## 5. 配置 DNS 记录

### 5.1 如果你的站点是 `username.github.io` 或 `orgname.github.io`

- 推荐使用 `CNAME` 记录：
  - 主机/名称：`www`（如果你使用 `www.example.com`）
  - 值/目标：`username.github.io`

- 如果你使用根域名 `example.com`：
  - 添加 4 条 A 记录，指向 GitHub Pages 的 IP 地址：
    - `185.199.108.153`
    - `185.199.109.153`
    - `185.199.110.153`
    - `185.199.111.153`

### 5.2 如果你的站点是仓库 Pages（不是 GitHub Pages 个人/组织主页）

- 使用 `CNAME` 记录，将你的自定义子域名指向 `username.github.io`。
- 根域名仍然需要添加上述 4 条 A 记录。

### 5.3 典型 DNS 配置示例

- `www.example.com` -> CNAME -> `username.github.io`
- `example.com` -> A -> `185.199.108.153`
- `example.com` -> A -> `185.199.109.153`
- `example.com` -> A -> `185.199.110.153`
- `example.com` -> A -> `185.199.111.153`

### 5.4 NameSilo 控制面板 DNS 配置步骤

1. 登录 NameSilo 后进入 `Domain Manager`。
2. 找到你购买的域名，点击右侧的 `Manage DNS` 或 `DNS Records`。
3. 如果你使用 `www.example.com`：
   - 添加一条 `CNAME` 记录。
   - `Host`/名称：`www`
   - `Value`/目标：`username.github.io`
   - `TTL` 保持默认即可。
4. 如果你希望根域名 `example.com` 也能访问：
   - 添加 4 条 `A` 记录，分别指向 GitHub Pages IP：
     - `185.199.108.153`
     - `185.199.109.153`
     - `185.199.110.153`
     - `185.199.111.153`
   - `Host`/名称 留空或填写 `@`，具体取决于 NameSilo 界面。
5. 保存 DNS 修改。

> 在 NameSilo 中不需要使用 URL Redirect，将域名直接通过 DNS 记录指向 GitHub Pages 即可。

## 6. 等待生效

DNS 修改后通常需要几分钟到数小时生效。你可以使用 `ping`、`dig`、`nslookup` 或在线 DNS 查询工具检查域名是否已解析到 GitHub Pages 提供的地址。

## 7. 常见问题与排查

- `404 Not Found`
  - 检查仓库根目录是否存在 `CNAME` 文件，内容是否正确。
  - 检查 GitHub Pages 设置中的自定义域名是否和 `CNAME` 内容一致。

- HTTPS 证书未生效
  - 先确认 DNS 已正确解析。
  - 进入 GitHub Pages 设置，确认已勾选 `Enforce HTTPS`。

- 域名仍然指向旧站点
  - 清除浏览器缓存。
  - 检查 DNS TTL 是否较大，等待缓存刷新。

## 8. 备注

- 如果你希望 `example.com` 和 `www.example.com` 都能访问，请在域名提供商处同时添加根域名 A 记录和 `www` CNAME 记录，另在 GitHub Pages 设置中以一个主域名为准。
- 不要在 `CNAME` 文件中添加 `https://` 或路径，仅写域名。

完成以上配置后，你的 GitHub Pages 站点就可以通过自定义域名访问。
